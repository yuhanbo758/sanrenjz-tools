# 三人聚智工具 (SanRenJZ Tools) 插件开发文档

## 1. 插件系统概览 (Plugin System Overview)

三人聚智工具的插件系统基于 Electron 的多进程架构设计。每个插件运行在独立的渲染进程中，通过预加载脚本 (`preload.js`) 与主应用进行安全的通信。

### 1.1 核心概念
- **独立进程**：每个插件拥有独立的 `BrowserWindow`，互不干扰。
- **安全通信**：通过 `IPC` (Inter-Process Communication) 调用主程序提供的能力。
- **生命周期**：插件可由搜索窗口、超级面板或全局快捷键唤起。
- **多入口支持**：插件可以注册多个功能 (`features`)，分别对应不同的触发指令。

---

## 2. 插件结构规范 (Plugin Structure)

标准的插件目录结构如下：

```text
sanrenjz-tools-plugin-name/
├── plugin.json          # [必需] 插件配置清单
├── index.html           # [必需] 插件主界面入口
├── preload.js           # [必需] 预加载脚本 (API 桥接)
├── logo.ico             # [必需] 插件图标 (Windows下任务栏显示)
├── logo.png             # [可选] 插件图标 (部分UI展示)
├── assets/              # [可选] 静态资源目录 (css, js, images)
└── README.md            # [推荐] 插件说明文档
```

### 命名规范
- **文件夹名称**：建议以 `sanrenjz-tools-` 开头，如 `sanrenjz-tools-password`。
- **插件名称**：在 `plugin.json` 中定义，应简短易懂。

## 3. 配置清单详解 (plugin.json)

`plugin.json` 是插件的元数据定义文件，决定了插件如何被系统加载和识别。

```json
{
  "pluginName": "插件显示名称",
  "description": "插件功能简介",
  "version": "1.0.0",
  "author": "开发者名称",
  "category": "效率工具",
  "main": "index.html",
  "logo": "logo.ico",
  "preload": "preload.js",
  "features": [
    {
      "code": "feature_unique_id",
      "explain": "功能简述",
      "description": "详细描述（用于超级面板提示）",
      "cmds": ["关键词1", "关键词2"],
      "icon": "logo.ico",
      "platform": ["win32", "darwin", "linux"],
      "mode": "none",
      "superPanel": true,
      "priority": 10
    }
  ],
  "pluginSetting": {
    "width": 1000,
    "height": 680,
    "single": true,
    "autoHideMenuBar": true
  }
}
```

### 关键字段说明

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `pluginName` | String | 插件在列表中的显示名称 |
| `main` | String | 入口 HTML 文件路径 |
| `preload` | String | 预加载 JS 文件路径 |
| `features` | Array | 插件提供的功能列表 |
| `pluginSetting` | Object | 窗口默认配置 |

### Feature 对象详解

- **code**: 功能唯一标识符，用于在代码中区分不同入口。
- **cmds**: 触发关键词数组。支持字符串或对象配置（正则/特殊类型）。
- **mode**: 
  - `"none"`: 独立窗口模式（最常用），插件接管完整的 UI 渲染。
  - `"list"`: 列表模式，配合 `window.exports` 的 `search` 钩子使用（类似 uTools）。
- **superPanel**: `true` 表示该功能可被添加到超级面板。

## 4. 预加载脚本与 API (Preload & API)

`preload.js` 是连接插件页面与底层系统的桥梁。为了安全性，建议使用 `contextBridge` 暴露 API，同时使用 `window.exports` 定义生命周期钩子。

### 4.1 推荐的 preload.js 模板

```javascript
const { contextBridge, ipcRenderer } = require('electron');

// 1. 暴露安全的 API 给渲染进程 (前端页面)
contextBridge.exposeInMainWorld('electronAPI', {
    // 数据存储
    storage: {
        get: (key) => ipcRenderer.sendSync('plugin-storage-get', '插件名称', key),
        set: (key, value) => ipcRenderer.sendSync('plugin-storage-set', '插件名称', key, value),
        remove: (key) => ipcRenderer.sendSync('plugin-storage-remove', '插件名称', key)
    },
    // 窗口控制
    window: {
        hide: () => ipcRenderer.invoke('hide-plugin-window', '插件名称'),
        close: () => ipcRenderer.invoke('close-plugin-window', '插件名称'),
        minimize: () => ipcRenderer.invoke('minimize-plugin-window', '插件名称'),
        maximize: () => ipcRenderer.invoke('maximize-plugin-window', '插件名称'),
        togglePin: () => ipcRenderer.invoke('toggle-plugin-pin-window', '插件名称')
    },
    // 系统对话框
    dialog: {
        showOpenDialog: (opts) => ipcRenderer.sendSync('show-open-dialog', opts),
        showSaveDialog: (opts) => ipcRenderer.sendSync('show-save-dialog', opts)
    },
    // 剪贴板与输入
    action: {
        copy: (text) => {
            const { clipboard } = require('electron');
            clipboard.writeText(text);
        },
        // 尝试插入内容到上一个活动窗口
        insert: (content) => ipcRenderer.invoke('handle-content-insertion', { content })
    }
});

// 2. 定义生命周期钩子 (供主程序调用)
window.exports = {
    "feature_unique_id": { // 对应 plugin.json 中的 feature.code
        mode: "none",
        args: {
            // 进入插件时触发
            enter: (action) => {
                console.log('Plugin entered:', action);
                // 可在此处执行初始化逻辑，例如根据 action.type 决定显示什么界面
            },
            // 搜索模式下的输入回调 (仅 mode: "list" 有效)
            search: (action, searchWord, callbackSetList) => {
                // 实现搜索逻辑并回调列表
            },
            // 列表项选择回调 (仅 mode: "list" 有效)
            select: (action, itemData) => {
                // 处理选中项
            }
        }
    }
};
```

### 4.2 可用 IPC 通道列表

主进程 (`main.js`) 提供了以下 IPC 通道供插件调用：

#### 同步通道 (`ipcRenderer.sendSync`)
- `plugin-storage-get`: 获取持久化数据
- `plugin-storage-set`: 保存持久化数据
- `plugin-storage-remove`: 删除数据
- `show-open-dialog`: 打开文件选择框
- `show-save-dialog`: 打开文件保存框

#### 异步通道 (`ipcRenderer.invoke`)
- `close-plugin-window`: 关闭窗口
- `minimize-plugin-window`: 最小化
- `maximize-plugin-window`: 最大化/还原
- `toggle-plugin-pin-window`: 切换钉住状态
- `handle-content-insertion`: 执行“复制并模拟粘贴”流程
- `hide-search-window`: 隐藏主搜索框
- `restore-previous-focus`: 尝试将焦点归还给之前的应用

## 5. 前端开发指南 (Frontend Development)

### 5.1 界面规范
由于插件运行在无边框窗口中，你需要自己设计标题栏（如果需要）或使用全屏布局。
- **HTML**: 标准 HTML5 结构。
- **CSS**: 推荐使用 Flexbox/Grid 布局，适配 `pluginSetting` 中定义的宽高。
- **JS**: 通过 `window.electronAPI` (在 preload 中定义的名称) 调用底层能力。

### 5.2 示例：在前端调用 API

```javascript
// index.html 中的 script

// 保存数据
document.getElementById('saveBtn').addEventListener('click', () => {
    const data = { theme: 'dark' };
    window.electronAPI.storage.set('settings', data);
});

// 关闭窗口
document.getElementById('closeBtn').addEventListener('click', () => {
    window.electronAPI.window.close();
});

// 复制内容
document.getElementById('copyBtn').addEventListener('click', () => {
    window.electronAPI.action.copy('Hello World');
});
```

## 6. 开发与调试 (Development & Debugging)

1.  **创建插件**：在 `app/software/` 下新建插件目录。
2.  **配置**：编写 `plugin.json`。
3.  **启动应用**：
    ```bash
    npm run dev
    ```
    此命令会开启 Electron 的调试端口 (9229)。
4.  **调试**：
    - 在插件窗口中，你可以像在 Chrome 中一样使用开发者工具 (DevTools)。
    - 如果插件窗口没有默认打开 DevTools，可以在代码中添加 `console.log`，日志会输出到启动应用的终端中。

## 7. 最佳实践 (Best Practices)

1.  **资源隔离**：尽量不依赖外部网络资源（CDN），将 CSS/JS 库打包在插件目录中，确保离线可用且加载速度快。
2.  **状态保存**：利用 `plugin-storage` API 保存用户的偏好设置或最后状态，提升体验。
3.  **错误处理**：在 `preload.js` 中包裹 `try-catch`，防止因 IPC 通信失败导致插件白屏。
4.  **响应式设计**：考虑到用户可能调整窗口大小（如果允许），界面应具备一定的响应式能力。
5.  **安全**：不要在 `preload.js` 中直接暴露 `ipcRenderer` 或 `remote` 对象，只暴露封装好的具体方法。

## 8. 常见问题 (FAQ)

**Q: 为什么我的插件在超级面板里不显示？**
A: 请检查 `plugin.json` 中对应的 feature 是否设置了 `"superPanel": true`，以及 `preload.js` 中是否正确导出了 `window.exports`。

**Q: 如何实现“复制后自动隐藏”？**
A: 调用复制 API 后，调用 `window.electronAPI.window.close()`。

**Q: 支持 Node.js API 吗？**
A: 主应用开启了 `nodeIntegration: true`，因此在插件的 `preload.js` 和 `index.html` (如果未开启 contextIsolation) 中可以直接使用 `require('fs')` 等模块。但在 `preload.js` 中封装并通过 `contextBridge` 暴露是更规范的做法。
