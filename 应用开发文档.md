# 三人聚智工具 (SanRenJZ Tools) 应用开发文档

## 1. 项目概览 (Project Overview)

### 1.1 目的与范围 (Purpose & Scope)
**三人聚智工具** 是一个基于 Electron 的可扩展桌面效率工具平台。其核心价值在于通过插件化架构，集成搜索、工具面板、文本处理、自动化脚本等多种功能，为用户提供统一、高效的操作入口。

**目标用户**：开发者、文字工作者、效率追求者。
**核心价值**：通过快捷键快速唤起工具，减少上下文切换，提升工作流效率。

### 1.2 系统能力 (System Capabilities)
- **插件系统**：支持动态加载 HTML/JS 插件，具备独立的进程/窗口管理。
- **多入口交互**：
  - **主窗口**：应用管理与插件浏览。
  - **搜索浮窗** (`Alt+Space`)：快速检索与执行功能 (类似 Spotlight/uTools)。
  - **超级面板** (`Ctrl+Alt+P`)：常用功能的一键访问面板。
  - **系统托盘**：后台常驻与快速菜单。
- **自动化支持**：模拟键盘输入、粘贴、窗口焦点控制。
- **数据持久化**：提供隔离的插件数据存储机制。

### 1.3 技术背景 (Technical Context)
系统采用 **Electron** 框架构建，利用其跨平台能力和 Node.js 运行时。架构上深受 uTools 等效率工具启发，采用 "主进程核心 + 渲染进程UI + 独立插件进程" 的模式。
- **主进程**：负责生命周期、全局快捷键、窗口管理、插件调度。
- **渲染进程**：负责用户界面展示。
- **插件环境**：提供 `Node.js` 集成环境 (`nodeIntegration: true`)，允许插件直接调用系统 API。

### 1.4 AI 相关性 (AI Relevance)
本项目结构清晰，模块化程度高，非常适合 AI 辅助开发：
- **插件化架构**：AI 可以生成独立的插件包（包含 `plugin.json` 和逻辑代码），而无需修改核心代码。
- **标准化接口**：`PluginManager` 提供了统一的 API，AI 易于理解和调用。
- **明确的文档**：本通过详细描述了上下文，有助于 AI 理解系统边界。

## 3. 架构与结构分析 (Architecture & Structure Analysis)

### 3.1 目录结构 (Directory Structure)

```text
sanrenjz-tools/
├── main.js                     # [核心] 主进程入口，负责生命周期、快捷键、窗口管理
├── app/
│   ├── software_manager.js     # [核心] 插件管理器，负责插件加载、执行、通信
│   └── software/               # [数据] 插件存放目录 (每个子目录为一个插件)
│       ├── sanrenjz-tools-text/         # 示例：文本片段助手
│       ├── sanrenjz.tools-ai/           # 示例：AI助手
│       └── ...
├── index.html                  # [UI] 主窗口界面 (插件列表)
├── search-window.html          # [UI] 搜索浮窗界面 (快捷启动)
├── super-panel.html            # [UI] 超级面板界面 (常用功能网格)
├── package.json                # [配置] 项目依赖与构建脚本
├── build/                      # [资源] 构建资源 (图标等)
└── custom-super-panel-actions-example.json  # [配置] 超级面板配置示例
```

### 3.2 模块边界 (Module Boundaries)

1.  **Main Process (`main.js`)**:
    - **职责**: 应用入口，初始化 `PluginManager`，管理 `BrowserWindow` 实例 (`mainWindow`, `searchWindow`, `superPanelWindow`)，注册 `globalShortcut`，处理 `Tray` 事件。
    - **对外接口**: 通过 `ipcMain` 监听渲染进程消息，通过 `@electron/remote` 暴露对象。

2.  **Plugin Manager (`app/software_manager.js`)**:
    - **职责**: 扫描 `app/software` 目录，解析 `plugin.json`，创建插件窗口，管理插件生命周期，处理插件数据存储。
    - **关键类**: `PluginManager`
    - **状态管理**: `pluginWindows` (Map), `pluginPinnedMap` (Map), `dynamicFeatures` (Map).

3.  **UI Renderers**:
    - **Main Window**: 展示已安装插件，提供管理入口。
    - **Search Window**: 接收用户输入，过滤插件/功能，发送执行指令。
    - **Super Panel**: 读取配置，展示快捷操作图标。

### 3.3 组件关系 (Component Relationships)
- **Main -> PluginManager**: 主进程实例化插件管理器，并在快捷键触发时调用其方法（如 `getTextSnippets`）。
- **Renderer -> Main**: 渲染进程通过 IPC 或 Remote 调用主进程功能（如隐藏窗口、退出应用）。
- **PluginManager -> Plugin Window**: 管理器创建并持有插件窗口实例，控制其显示/隐藏/销毁。

## 4. 核心逻辑与算法 (Core Logic & Algorithm Documentation)

### 4.1 插件加载机制
1.  **扫描**: `PluginManager.getPluginList()` 遍历插件目录。
2.  **解析**: 读取每个目录下的 `plugin.json`。
3.  **清洗**: 对 `features.cmds` 等字段进行序列化清洗，防止 IPC 传输错误。
4.  **注册**: 将插件信息加载到内存，供搜索窗口调用。

### 4.2 插件执行逻辑 (`runPlugin`)
- **输入**: 插件路径 (`pluginPath`)，功能参数 (`features`)。
- **判断类型**:
  - **HTML 插件**:
    1.  检查 `pluginWindows` 是否已有实例。
    2.  若有，激活并置顶。
    3.  若无，创建新 `BrowserWindow` (无边框, `nodeIntegration: true`)。
    4.  设置 AppUserModelID (Windows) 以确保任务栏图标独立。
    5.  若指定 `feature.code`，则通过 IPC 通知插件执行特定动作。
  - **JS 插件**:
    - 在 Node.js 环境中直接执行脚本逻辑 (目前支持有限)。

### 4.3 窗口钉住逻辑 (Pinning)
- **状态**: `pluginPinnedMap` (Map<pluginName, boolean>)。
- **触发**: `Ctrl+D` (默认)。
- **行为**:
  - **未钉住**: 窗口 `blur` 事件触发后 500ms 自动隐藏。
  - **已钉住**: 窗口 `blur` 事件忽略隐藏操作，保持常驻。

### 4.4 数据持久化 (Data Persistence)
- **存储位置**: 用户数据目录下的 `plugin-data/` (可配置)。
- **格式**: JSON 文件 `<PluginName>-storage.json`。
- **API**:
  - `getPluginStorageItem(pluginName, key)`
  - `setPluginStorageItem(pluginName, key, value)`
  - 自动处理 JSON 序列化与反序列化。

## 5. 数据结构与 API 规范 (Data Structures & API Specifications)

### 5.1 插件清单 (`plugin.json`)
```json
{
  "pluginName": "插件名称",
  "description": "插件描述",
  "version": "1.0.0",
  "author": "作者",
  "category": "工具",
  "main": "index.html",  // 入口文件
  "preload": "preload.js", // 预加载脚本 (可选)
  "logo": "logo.ico",    // 图标
  "features": [          // 功能列表
    {
      "code": "feature_code",
      "explain": "功能解释",
      "cmds": ["关键词1", "关键词2"] // 触发关键词
    }
  ],
  "pluginSetting": {     // 窗口配置
    "width": 800,
    "height": 600
  }
}
```

### 5.2 插件内部可用 API (通过 Remote/IPC)
由于开启了 `nodeIntegration`，插件可以直接使用 Node.js API (`fs`, `path`, `child_process`)。
此外，`utools` 风格的 API (如 `utools.onPluginEnter`) 目前通过 `preload.js` 或直接在 `index.html` 中实现逻辑适配。

## 6. 环境与部署指南 (Environment & Deployment Guide)

### 6.1 开发环境搭建
1.  **前置要求**: Node.js v16+, npm v8+。
2.  **安装**:
    ```bash
    git clone <repository_url>
    cd sanrenjz-tools
    npm install
    ```
3.  **运行**:
    ```bash
    npm start      # 普通启动
    npm run dev    # 调试模式 (开启 DevTools 端口)
    ```

### 6.2 构建流程
使用 `electron-builder` 进行打包：
- **Windows**: `npm run build-win` (生成 .exe / .nsis)
- **macOS**: `npm run build-mac` (生成 .dmg / .app)
- **Linux**: `npm run build-linux` (生成 AppImage / .tar.gz)
- **产物目录**: `dist/`

### 6.3 环境变量与配置
- **配置文件**: `userData/settings.json`
- **主要配置项**:
  - `globalHotkey`: 全局唤起快捷键 (默认 `Alt+Space`)
  - `autoStart`: 开机自启
  - `customPluginDataPath`: 自定义数据存储路径

## 7. 扩展开发指南 (Extension Development Guide)

### 7.1 插件开发模式
1.  在 `app/software/` 下创建新文件夹。
2.  创建 `plugin.json` 定义元数据。
3.  创建 `index.html` 作为 UI 入口。
4.  (可选) 创建 `preload.js` 进行预加载逻辑。

### 7.2 最佳实践
- **资源隔离**: 尽量将插件所有资源放在自身目录下。
- **状态保存**: 使用 `pluginManager` 提供的存储 API，避免污染全局环境。
- **UI 规范**: 建议使用无边框设计，自定义标题栏，以保持与主应用风格一致。
- **错误处理**: 确保插件内的报错不会导致主进程崩溃，使用 `try-catch` 包裹关键逻辑。

### 7.3 调试
- 在 `npm run dev` 模式下，可以通过主窗口或插件窗口的控制台 (Console) 查看日志。
- 主进程日志会输出到终端。
